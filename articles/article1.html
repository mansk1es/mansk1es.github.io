LDAP Relay

What is LDAP?

LDAP (Lightweight Directory Access Protocol) is a protocol that allows everyone (in the network) to locate data in the network. Its main use is for authentication, it can be used to query information in the network, and collect important information about it (users, groups, OUs, etc.).

Taking advantage of the LDAP protocol.

To make LDAP Relaying work, the most important requirement we need is the "Require LDAP Signing" option disabled on the Domain Controllers! This is very common, because the LDAP Signing option is turned off by default.

Unauthenticated Domain Takeover

Relaying LDAP authentication over the network does not require the attacker to have valid, compromised credentials of a domain user. The whole essence and nature of the attack, is to take advantage of normal (NTLM challenge-response) authentication over the network, by forcing domain objects (users, computers) to authenticate to the attacker instead of the DC directly, the attacker then forwards (relays) valid LDAP authentication to the Domain Controller on behalf of the captured user/machine authentication. Using the valid authentication, we are able to practically dump information about the domain (and potentially importing it to bloodhound to have an even bigger picture), fully compromise computers in the network (Resource Based Constrained Delegation), adding new machine accounts to the Domain (and elevating them), and even directly take over the entire domain.

Requirements

1. LDAP Signing is not enabled on the Domain Controllers.

2. ntlmrelayx.py alongside with MitM6 and/or Responder (to catch NTLM hashes).

3. For certain LDAP Relay attacks (such as Resource Based Constrained Delegation), the Domain Functionality Level has to be 2012 and above.

4. NTLM authentication enabled.
